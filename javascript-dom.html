Describe JavaScript and its relationship to HTML and CSS.

Imagine Times Square, we can turn this into a system described by these three coding languages. Imagine just the buildings no lights no billboards and people just standing in grey clothes. This is the HTML part of Times Square just blank buildings and blank humans. It looks like Times Square but isn’t quite the same and doesn’t look alive.

Here comes the CSS to make Times Square look great. Now that we have the buildings and the billboards and the people we can start to add some colour. The buildings get their logos with flashy neon glows the people now each have their own fashion style and can be identified as different from the next the billboards contain images and can be positioned on hte buildings where needed. Times square is certainly looking good now but people still aren’t moving and nothing feels like it’s alive.

In comes JavaScript(JS) the billboards now play videos and some are in sync, logos jump to life. The people now seem like people with needs and wants they start moving throughout the city. Taxi’s start coming and stop if someone calls them. The city suddenly feels like it is interactable and we can have an impact by being involved now.

This is HTML, CSS and JS. HTML is the building foundations holds the information, CSS covers the style and looks, JS controls functionality it makes the city interactive.

Explain control flow and loops using an example process from everyday life

--Control Flow--

Control flow is the order which the computer reads its instructions(code). If we we’re to write a list of a tasks before leaving the house it could consists of -

  • Eat
  • Shower
  • Grab keys
  • Drive

The order that we wrote this down would be the order to complete these tasks in. Control flow works the same way and is useful to make sure the computer does the correct job at the correct time. It will also break if certain tasks can’t be done before hand. Using the previous example if our list looked like this.

  • Eat
  • Shower
  • Drive
  • Grab keys

The computer will shoot an error, rightfully so it would be impossible to drive without keys. Control flow is important not only for the order to complete tasks in but to make sure that those tasks can actually be done based of what has happened beforehand.

--Loops--

Loops are the computers way of counting. Say we follow the tasks above and at the eat part we decide to eat cereal. Say we know it is going to take twenty spoonfuls to finish the cereal. Instead of saying the same thing over and over again 20 times “Eat a spoonful, Eat a spoonful, Eat a spoonful…” we could tell the computer to do the same task 20 times in two lines of code vs 20, this is more efficient and useful. It won't work for lunch as it could take more or less “mouthfuls” so to make it work for lunch lets say do this task below until the plate is empty and now it works for every meal not only lunch.

What is the difference between accessing data from arrays and objects?

Arrays and object have differnt way to access the info inside. It has a lot to do with how the info id stored. Arrays have a index when things are stored in arrays they have numbered positions called an index and to access items you call on its position in the array not on the item. So if the array had a string/word stored at the 4th position since we count form zero we would use array[3] to get that item out. Objects can be called by the individual items using dot notation or bracket notation. Objects store values using a key so if there was an object called names taht stored info by names and we wanted a specific person unlike arrays we don’t need its position we need the name so we call names.john for example or using bracket notation names[“john”]

Explain what functions are and why they are helpful.

Functions are used to write a piece of code once and have the ability to call on it later. There are many use cases for this like the buttons above I want them to do something when pressed. If I was to write out the code not inside a function like we say in control flow the computer will read the document and complete the code. If it is in a function it will not read this code until it sees a peice of called calling the function to run. Then it will go back and read the function and complete the code. Going back to the buttons I wrote two functions and called upon them when the buttons were pressed. The functions can be run an infinite number of times as long as someone is there to press the buttons.

Functions can also be used to avoid writing bits of code over and over again but instead writing it once and calling upon that function when it is needed. Not only can they be used to execute the same peice of code over and over they are at a higher level used to complete the same tasks with different inputs using parameters (something you put in the function), for example if you needed lots of numbers added together a lot in a programme you wouldn’t want to do the math over and over you could write a function that is similar to an algebraic expression. Using X and Y you could tell the function to add these together but when you call on the function you give it real numbers in place of x and y and it will replace the numbers exactly like algebra.This can grow to do very complex problems which saves effort and time.
